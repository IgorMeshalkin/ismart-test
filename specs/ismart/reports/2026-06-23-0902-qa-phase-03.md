Risks:
- `MediaRecorder` and `getUserMedia` require a secure context (HTTPS or localhost); production deployments must serve the frontend over HTTPS for voice recording to function.
- `MediaRecorder` MIME type varies by browser (`audio/webm` on Chromium, `audio/mp4` on Safari); the implementation reads `recorder.mimeType` at stop time which correctly adapts to the browser.
- The auth layout guards use `useEffect` for redirect, which means a brief render of `null` occurs before redirect. This is the standard Next.js App Router pattern for client-side auth checks without a middleware.

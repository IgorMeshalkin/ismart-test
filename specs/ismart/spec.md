# iSmart Product Specification

## 1. Описание продукта

iSmart — платформа для записи аудио, автоматической расшифровки речи в текст и организации полученных материалов в базы знаний.
Пользователи могут записывать аудио в браузере или загружать готовые аудиофайлы, получать транскрипции, 
группировать материалы в базы знаний и предоставлять доступ другим пользователям.

---

## 2. Цели продукта

Основные цели системы:

- преобразование аудио в текст;
- хранение и организация транскрипций;
- совместная работа с базами знаний;
- контроль использования через тарифные планы;
- поддержка дальнейшего масштабирования продукта.

---

## 3. Роли пользователей

Система поддерживает две роли пользователей:

- USER — обычный пользователь;
- ADMIN — администратор.

### Пользователь

Может:

- регистрироваться и авторизовываться;
- управлять своим профилем;
- записывать аудио;
- загружать аудиофайлы;
- просматривать историю транскрипций;
- создавать базы знаний;
- управлять своими базами знаний;
- предоставлять доступ другим пользователям к базам знаний;
- получать уведомления.

### Администратор

Может:

- управлять тарифными планами;
- управлять пользователями;
- управлять подписками;
- просматривать статистику использования системы.

---

## 4. Основные возможности системы

### Аутентификация

Система должна поддерживать:

- регистрацию пользователей;
- вход в систему;
- выход из системы;
- восстановление доступа к аккаунту.

### Работа с аудио

Пользователь должен иметь возможность:

- записывать аудио через браузер;
- загружать существующие аудиофайлы;
- просматривать статус обработки аудио;
- просматривать историю своих файлов.

### Транскрипция

Пользователь должен иметь возможность:

- отправлять аудио на обработку;
- отслеживать статус обработки через polling;
- просматривать результат транскрипции;
- скачивать результат транскрипции.

В MVP Frontend получает обновления статуса транскрибации через периодический polling API.

### Тарифные планы

Система должна поддерживать:

- различные тарифные планы;
- лимиты на обработку аудио;
- лимиты на хранение данных;
- подписки пользователей;
- покупку нескольких тарифных планов на будущие периоды.

### Базы знаний

Пользователь должен иметь возможность:

- создавать базы знаний;
- редактировать базы знаний;
- удалять базы знаний;
- добавлять транскрипции в базы знаний;
- удалять транскрипции из баз знаний;
- предоставлять доступ другим пользователям;
- делать базу знаний публичной или приватной.

### Уведомления

Система должна уведомлять пользователей о:

- завершении транскрипции;
- приглашении в базу знаний;
- приближении окончания подписки;
- других важных событиях системы.

---

## 5. Пользовательские сценарии

### Сценарий 1. Запись и расшифровка аудио

1. Пользователь начинает запись аудио.
2. Пользователь завершает запись.
3. Система сохраняет аудиофайл.
4. Система запускает обработку.
5. Пользователь получает уведомление о завершении.
6. Пользователь просматривает результат транскрипции.

### Сценарий 2. Загрузка готового аудиофайла

1. Пользователь выбирает аудиофайл.
2. Система проверяет доступные лимиты.
3. Файл загружается в систему.
4. Система запускает обработку.
5. Пользователь получает результат транскрипции.

### Сценарий 3. Создание базы знаний

1. Пользователь создаёт базу знаний.
2. Пользователь добавляет транскрипции.
3. Система связывает материалы с базой знаний.
4. Пользователь получает доступ к собранным материалам.

### Сценарий 4. Предоставление доступа

1. Владелец базы знаний приглашает другого пользователя.
2. Система предоставляет доступ.
3. Приглашённый пользователь получает уведомление.
4. Приглашённый пользователь получает доступ к базе знаний.

---

## 6. Бизнес-правила

### Лимиты использования

Пользователь не может запускать обработку аудио при отсутствии достаточного количества доступных секунд обработки.

При создании File система резервирует доступные секунды обработки согласно длительности аудио.

После успешного завершения обработки система снимает резерв и окончательно списывает секунды обработки.

При ошибке обработки система снимает резерв без окончательного списания секунд.

### Ограничения хранения

Пользователь не может превышать доступный лимит хранилища согласно активной подписке.

При создании File система резервирует доступный лимит хранилища согласно размеру аудиофайла.

После успешного завершения обработки система снимает резерв и окончательно списывает лимит хранилища.

При ошибке обработки система снимает резерв без окончательного списания лимита хранилища.

### Очередь тарифных планов

Пользователь может купить несколько тарифных планов на будущее.

Если у пользователя уже есть активный тарифный план, новый купленный план становится будущим и активируется только после окончания текущего активного плана.

В один момент времени у пользователя может быть только один активный тарифный план.

В MVP неиспользованные лимиты не переносятся из одного тарифного плана в следующий.

### Доступ к данным

Пользователь имеет доступ только:

- к собственным материалам;
- к публичным базам знаний;
- к приватным базам знаний, к которым ему предоставлен доступ.

### Публичные и приватные базы знаний

База знаний может быть публичной или приватной.

Публичная база знаний доступна всем пользователям для просмотра.

Приватная база знаний доступна только владельцу и пользователям, которым предоставлен доступ.

### Владение контентом

Владелец базы знаний управляет доступом к ней и её содержимому.

---

## 7. MVP

В первую версию входят:

- регистрация и авторизация;
- запись аудио;
- загрузка аудиофайлов;
- транскрипция;
- управление тарифами;
- покупка нескольких тарифных планов на будущие периоды;
- базы знаний;
- публичные и приватные базы знаний;
- предоставление доступа к базам знаний;
- уведомления внутри системы.

---

## 8. Что не входит в MVP

- интеграция с платёжными системами;
- SMS-уведомления;
- Telegram-уведомления;
- поиск по содержимому через ИИ;
- перенос неиспользованных лимитов из одного тарифного плана в следующий;
- публичное API;
- мобильные приложения.

---

## 9. Критерии готовности MVP

MVP считается готовым, если:

- пользователь может загрузить аудио;
- пользователь может получить транскрипцию;
- лимиты подписки корректно контролируются;
- базы знаний поддерживают совместный доступ;
- система создаёт уведомления по ключевым событиям.
